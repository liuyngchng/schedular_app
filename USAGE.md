# 定时启动器 — 使用指南

## 概述

定时启动器可以让你在指定的时间，自动打开手机上的任意应用。每天两个时段，到点即开。

---

## 前置条件（重要）

要使定时启动器在锁屏/息屏状态下，到达设定时间后顺利打开目标 App，必须完成以下设置：

### 必须满足

| 条件 | 说明 |
|------|------|
| **关闭锁屏密码** | 如果有 PIN/图案/指纹等锁屏密码，目标 App 会被启动在锁屏**后面**，无法实现"到时间直接可用"。手机应设为无锁屏密码（滑动解锁或无锁屏）。 |
| **开启无障碍服务** | Android 12+ 严格限制后台应用启动 Activity。开启无障碍服务后，定时启动器拥有最高系统优先级，可以绕过此限制。**这是最关键的一步。** |
| **授予精确闹钟权限** | Android 12+ 必须手动授予，否则闹钟无法触发。 |
| **关闭省电模式** | 省电模式/Doze 会延迟或阻止后台闹钟触发。 |
| **首次启动后手动打开过 App** | Android 12+ 要求应用至少被手动启动过一次，否则 `BOOT_COMPLETED` 广播不会生效，重启后闹钟会丢失。 |

### 强烈建议

| 条件 | 说明 |
|------|------|
| **加入电池优化白名单** | 设置 → 应用 → 定时启动器 → 电池 → 不优化。防止系统在后台限制 App，提高闹钟准点率。 |
| **授予悬浮窗权限** | 允许 App 在锁屏界面上方绘制窗口，增加一层启动保障。 |

### 不需要

| 条件 | 说明 |
|------|------|
| 屏幕常亮 | 闹钟触发时会自动点亮屏幕（`setTurnScreenOn` + wakelock），无需手动保持亮屏。 |
| 目标 App 在前台 | 无需提前打开或保持目标 App 在后台运行。 |
| 本 App 保持在前台 | 定时启动器可以在后台运行，到时间自动触发。 |

---

## 第一步：开启无障碍服务

**这是最关键的一步。** 不开启则无法在息屏/后台状态下启动目标 App。

1. 打开 App，查看首页顶部的自检卡片
2. 找到「无障碍服务：未开启」→ 点击下方「去开启无障碍」
3. 在系统设置中找到「定时启动器」，打开开关
4. 确认对话框中点击「允许」

---

## 第二步：授予精确闹钟权限

首次打开应用时，系统会弹出权限请求对话框：

- 点击 **允许**
- 如果误点了拒绝，可前往 **系统设置 → 应用 → 定时启动器 → 闹钟和提醒** 手动开启

---

## 第三步：授予悬浮窗权限（建议）

1. 在自检卡片中找到「悬浮窗权限：未授予」→ 点击下方「去开启悬浮窗」
2. 打开「允许显示在其他应用的上层」

---

## 第四步：加入电池优化白名单（建议）

1. 首次保存设置时，App 会弹出电池优化豁免请求
2. 点击 **允许**
3. 或手动前往 **系统设置 → 应用 → 定时启动器 → 电池 → 不优化**

---

## 第五步：选择目标应用

1. 在首页点击 **选择应用** 卡片
2. 在弹出的列表中浏览已安装的 App
3. 点击你想定时启动的应用名称

> 列表会自动排除"定时启动器"自身。

---

## 第六步：设置启动时间

1. 点击「时段一」或「时段二」标签切换要编辑的时段
2. 点击时间区域，弹出时间选择器
3. 上下滑动选择小时和分钟（24 小时制）
4. 点击 **确定** 完成

> 两个时段独立设置，互不影响。如果只需一个时段，保留一个即可。

---

## 第七步：启用并保存

1. 打开 **启用定时** 开关
2. 点击底部 **保存** 按钮
3. 屏幕提示"设置已保存"，表示完成

> 每天自动执行，无需选择日期。关闭开关后闹钟会被取消。

---

## 自检功能

首页顶部会显示自检卡片，实时诊断以下状态：

- 精确闹钟权限
- 电池优化白名单
- 锁屏状态
- 目标应用是否已安装
- 闹钟是否已注册
- 无障碍服务是否开启
- 悬浮窗权限是否授予

绿色 ✓ 表示正常，红色 ✗ 表示需要处理。长按自检卡片可重新检测。未开启的无障碍服务或悬浮窗权限下方会有快捷入口按钮。

---

## 测试功能

在时段选择行右侧，有一个 **测试** 按钮：

- 点击后会立即手动启动你选择的目标 App
- 用于验证目标应用是否能正常打开
- 不会影响已设置的定时任务

---

## 执行日志

1. 在「启用定时」卡片右侧，点击 **执行日志**
2. 展开后显示历史执行记录，包含应用名和触发时间
3. 长按日志区域可弹出「清除日志」确认框
4. 再次点击 **执行日志** 收起

日志标签说明：
- `[闹钟触发]` — 闹钟到点，开始启动流程
- `[启动失败]` — 启动过程中出现异常
- `[应用不存在]` — 目标 App 可能已被卸载

---

## 常见问题

### 到了设定时间应用没有启动？

1. 打开 App 查看自检卡片，确认所有项目都是绿色 ✓
2. 最关键：确认无障碍服务已开启
3. 确认「启用定时」开关已打开
4. 确认手机未开启省电模式

### 闹钟准时吗？

使用 `setAlarmClock()` 精确闹钟，无随机偏移，应在设定时间 ±1 秒内触发。

### 重启手机后还需要重新设置吗？

不需要。应用会在开机后自动重新注册所有闹钟。

### 更换目标应用后需要重新设置时间吗？

不需要。更换应用不影响已有的时间设置，直接保存即可。
